LUXAGRID MASTER AI SYSTEM — PRO PACK

What this adds vs the basic pack:
- Stronger guardrails (Scope Contract + Protected Zones)
- A repeatable Branch + Verify + Log workflow
- Pre-made prompts for: bootstrap, stabilize, feature, bugfix, PR review, release check
- A Known Issues log to stop repeating the same fixes
- A simple check script (bash) and Windows PowerShell equivalent
- Git helper notes and a “safe edit” policy

--------------------------------------------------------------------
RECOMMENDED REPO SETUP (per project)
--------------------------------------------------------------------
Create these folders/files in every repo (copy from this PRO pack):

/ai
  MASTER_PROMPT.md
  SCOPE_CONTRACT.md
  AI_CHANGELOG.md
  KNOWN_ISSUES.md
  SESSION_NOTES.md
  PROMPTS/
    00_BOOTSTRAP.txt
    10_STABILIZE.txt
    20_FEATURE.txt
    30_BUGFIX.txt
    40_PR_REVIEW.txt
    50_RELEASE_CHECK.txt

/docs
  ARCHITECTURE.md

/scripts
  check.sh
  check.ps1

/.env.example

--------------------------------------------------------------------
ONE TIME (NEW APP)
--------------------------------------------------------------------
1) Copy the files above into your repo.
2) Run PROMPTS/00_BOOTSTRAP.txt in Claude Code.
3) Then work in micro-scopes using PROMPTS/20_FEATURE.txt.

--------------------------------------------------------------------
ONE TIME (50% DONE APP)
--------------------------------------------------------------------
1) Copy the files above into your repo.
2) Run PROMPTS/10_STABILIZE.txt in Claude Code.
3) Only after checks pass, start new features.

--------------------------------------------------------------------
DAILY LOOP (THE WHOLE SYSTEM)
--------------------------------------------------------------------
1) Create branch:
   feat/<name> OR fix/<name>

2) Paste ONE prompt:
   /ai/PROMPTS/20_FEATURE.txt OR /ai/PROMPTS/30_BUGFIX.txt
   Fill in the placeholders.

3) Require verification:
   - scripts/check.sh (mac/linux) OR scripts/check.ps1 (windows)
   - or closest npm scripts (lint/typecheck/build)

4) Require logging:
   - Update /ai/AI_CHANGELOG.md
   - Add any recurring issues to /ai/KNOWN_ISSUES.md

5) Merge only when checks pass.

--------------------------------------------------------------------
RULE THAT STOPS THE LOOP
--------------------------------------------------------------------
One change. One purpose. One verification. One log entry. One commit.
